# Интеграция CRM с сервисом MadeTask

Введение: данный документ описывает требования интеграции CRM и MadeTask для автоматизации управления задачами и проведения выплат исполнителям  

Цели:  
- учесть всех исполнителей (и внутренних, и внешних) в одной системе
- автоматизировать выплаты дизайнерам при выполнении задач
- снизить нагрузку на бухгалтерию (избежать ручных переводов)
  
Задачи:   
-	реализовать интеграцию CRM с MadeTask
-	автоматизировать процесс проведения выплат исполнителям
-	обеспечить синхронизацию данных между системами
-	автоматизировать процесс создания и управления задачами

1. Анализ текущих процессов
   
Текущий процесс:  
- менеджер ставит задачу дизайнерам через мессенджеры;
- чтобы проверить выполненную задачу, менеджеру необходимо найти её в чат;
- оплата производится вручную. Бухгалтерия переводит деньги через СБП, сверяя реквизиты в Excel.
  
Проблемы и причины  
- задачи теряются, потому что нет единого трекера и все задачи передаются через мессенджеры;
- много времени уходит на перевод денег, потому что бухгалтер вручную переводит деньги через СПБ, сверяя реквизиты в Excel;
- риск, что платеж может уйти не тому получателю, за счет ручного ввода данных;
- нет прозрачности в процессах, так как для проверки исполнения платежа необходимо ждать отчет из банка.

2. Проект будущего решения
   
Процесс после интеграции:  
1.	Менеджер создает задачу в CRM и назначает исполнителем дизайнер
2.	Дизайнер в CRM получает задачу и после ее выполнения меняет статус на “Готово”
3.	Менеджер проверяет выполненную работу 
4.	В случае, если задача выполнена верно, менеджер принимает ее и CRM направляет запрос на оплату в MadeTask
5.	В MadeTask проверяется наличие счета исполнителя:
- если счет привязан, производится автоматическая оплата и в CRM сохраняется информация о выплате
- если счет не привязан исполнителю приходит уведомление с запросом привязать счет 
6. Если менеджер не принял задачу дизайнера, менеджер направляет задачу на доработку и процесс повторяется
  
Роли пользователей и их сценарии:  
Менеджер – создает задачи в CRM, назначает исполнителей, проверяет выполненную работу, принимает задачу или отклоняет ее, отправляя на доработку.  
Дизайнер - получает задачи в CRM, выполняет их, изменяет статус на «Готово», при необходимости привязывает счет в MadeTask для получения выплат.

4. Работа с API MadeTask

4.1 Создание задачи: 
POST v2/tasks

Основные параметры: 

| Параметр | Тип | Описание |
|----------|----------|-----------------|
name | string	 | наименование задачи, необязательный параметр |
description	| string |	информативное описание задачи. Минимальное количество символов = 10 |
expired_at	| datetime	срок завершения задачи, больше или равен текущий день плюс один, даты передаются в формате ISO8601, необязательный параметр |
cost |	float	| стоимость задачи, сумма, которую получит исполнитель за выполненную задачу |
currency |	string |	валюта (RUB, USD, EUR) |
contractor_id |	integer |	уникальный номер исполнителя |
category_id |	integer |	уникальный номер категории задачи из справочника, обязательный параметр |
service_id |	integer |	уникальный номер услуги из справочника, обязательный параметр |
external_id	| string |	идентификатор в системе заказчика, необязательный параметр |

Пример запроса: 
{
   "name": "Необязательное наименование",
   "description": "Информативное описание задачи",
   "expired_at": "2022-05-22T00:00:00",
   "cost": 5000.00,
   "currency": "RUB",
   "contractor_id": 15,
   "category_id": 1,
   "service_id": 2,
   "external_id": "9c4f3ac6-6005-11ed-9b6a-0242ac120002"
}
Код успешного выполнения: 200 OK
Код неуспешного выполнения: 
400 – Bad Request
404 – Not found 
500 - Internal Server Error


4.2 Изменить статус на «Выполнено»
PATCH /tasks/{task_id}/update 
Основные параметры: 

| Параметр | Тип | Описание |
|----------|----------|-----------------|
task_id	integer	уникальный идентификатор задачи |
name	| string |	название задачи |
description	| string |	описание задачи |
expired_at |	datetime | срок завершения задачи, больше или равен текущий день плюс один, даты передаются в формате ISO8601 |
is_need_report |	bool	| нужен ли отчет по задаче |
cost	| float |	стоимость задачи, сумма, которую получит исполнитель за выполненную задачу (обновление доступно только для неоплаченных задач) |
currency	| string	| валюта (RUB, USD, EUR) (обновление доступно только для неоплаченных задач) |
external_id |	string |	идентификатор в системе заказчика, необязательный параметр |

Пример запроса: 
{
  "name": "Обновление сайта",
  "description": "Поменять дизайн главной страницы сайта",
  "expired_at": "2026-07-27T18:00:00Z",
  "is_need_report": true,
  "cost": 100000,
  "currency": "RUB",
  "external_id": "CRM-5"
}
Код успешного выполнения: 200 OK
Код неуспешного ответа: 
400 - Bad Request
404 - Not Found
409 – Conflict

5.	Требования 

5.1 Функциональные требования: 
ФТ-1: менеджер должен иметь возможность создавать задачи в CRM;
ФТ-2: менеджер должен иметь возможность принимать задачи или возвращать на доработку;
ФТ-3: если менеджер отклонил задачу, система должна отправить задачу на доработку исполнителю;
ФТ-4: система должна отображать все задачи в CRM;
ФТ-5: исполнитель должен получать задачи в CRM;
ФТ-6: исполнитель должен иметь возможность менять статус задачи на “Готово”;
ФТ-7: после принятия задачи менеджером, система автоматически должна инициировать оплату через MadeTask;
ФТ-8: система должна сохранить информацию о проведенном платеже;
ФТ-9: если счет исполнителя не привязан для оплаты, система отправляет уведомление с просьбой привязать счет;

5.2 Нефункциональные требования:
НФТ-1: уведомление исполнителю о необходимости привязать счет должно быть отправлено не позднее чем через 5 секунд после получения ответа от MadeTask;
НФТ-2: информация о платежах в системе должна храниться не менее 5 лет 
НФТ-3: обмен данными между CRM и MadeTask должен осуществляться в формате JSON 
НФТ-4: интеграция должна обеспечивать отсутствие дублирования данных между CRM и MadeTask 
НФТ-5: система должна обеспечивать разграничение прав доступа в соответствии с ролью пользователя
6.	Задачи для разработчиков

Use story 
1.	Как менеджер, я хочу передавать и создавать задачи через CRM, чтобы хранить информацию в одном месте и контролировать выполнение задач.
2.	Как менеджер, я хочу иметь возможность отправлять задачу на доработку через CRM, чтобы дизайнер видел актуальную задачу по доработке.
3.	Как дизайнер, я хочу получать задачи в CRM, чтобы видеть полный список задач.
4.	Как дизайнер, я хочу менять статус задачи в CRM, чтобы менеджер видел исполнение задач.
5.	Как менеджер, я хочу получать информацию о статусе выплаты, чтобы контролировать оплату за выполненную работу.

Задача 1: реализовать создание задачи в CRM
Описание: необходимо реализовать возможность создания задач менеджером через CRM. Менеджер должен иметь возможность дать название задаче, заполнить описание задачи, назначить исполнителя и срок выполнения задачи. Далее задача сохраняется и отображается в списке задач.

Задание 2: реализовать отправку задачи на доработку
Описание: необходимо реализовать возможность отправки задач на доработку через CRM, чтобы дизайнер видел актуальную задачу с описанием по доработке.

Задание 3: реализовать отображение списка задач
Описание: необходимо реализовать отображение списка задач в CRM, чтобы видеть: название задачи, статус, исполнитель и срок выполнения задачи.

Задание 4: реализовать изменение статуса задач
Описание: необходимо реализовать возможность изменения статуса задач дизайнером в CRM, далее информация о статусе должна отображаться у менеджера 

Задание 5: реализовать получение информации о платежах
Описание: необходимо реализовать сохранение и отображение информации о проведении платежей, чтобы контролировать оплаты выполненных работ

Задание 6: реализовать синхронизацию CRM и MadeTask 
необходимо реализовать обмен данными между CRM и MadeTask для автоматической передачи информации о задачах, статусах выполнения и выплатах исполнителям.
